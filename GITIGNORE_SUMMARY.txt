⚠️  IMPORTANT NOTES

1. .env files are IGNORED (correct for secrets)
   → Commit .env.example instead with dummy values
   → Team members copy .env.example → .env locally

2. package-lock.json is IGNORED
   → This is optional but recommended for monorepos
   → Team can regenerate with: npm ci

3. .husky/ is IGNORED
   → Hooks are installed locally via: npm install
   → Pre-commit/pre-push hooks still work locally

4. coverage/ is IGNORED
   → Generate locally: npm run test:coverage
   → Don't commit coverage reports

5. dist/ and build/ are IGNORED
   → Generate locally: npm run build
   → CI/CD builds on deployment
